The blocks

Tell your non-tech friend a story about a time you: were blocked on a simple problem. elegantly solved a problem.

In both cases, explain: What was the problem? What problem-solving techniques did you use? How did you feel throughout the process? What did you learn?

In sprint 3, I had a couple of blocks while doing the JavaScript Challenge. The first time it happened was in Challenge 5 - Conditionals, Q1 & Q2, the if...else statement.
After I write the code, I tried console.logging it and check if the code is working and it did. But when I run the check sheet, it won't pass. The erro message does not provide enough information for me to investigate. I googled to see how other people solved the prolem and tried rewrite my code in a different way but the same thing happened. So one day after the home group meeting, I asked the facilitator to check the code for me. We had a coaching call and I shared my screen. In the end, it was the code that's the problem, it was the function name. I used the same function name for both challenges, that's why it won't pass the tests. As soon as I changed the name of the challenge, the problems solved.

Another block happened in Challenge 7 - Loops Q4 & Q5. I tried all the techniques I used previouly and nothing worked. I wrote pseudocode, googled, reading error message, re-writing the code in a different way, try to read through on the discord channel if others had the same problems and how they solved the problems. I can see that many students were stuck on these questions, after all, they are stretch materials. But nobody had the same problem I had. So I asked for help on the discord channel. A fellow student who had the same problem replied to my message. The answer didn't solve my problem immediately but it pointed me to a different direction. I sat on it for a day and carried on with other tasks. I tried again the next day, still not working. I pondered for another day. I think it was the third day that it dawned on me that I finally understood what the solution is. So I tried a different way to write the code. And sure enough, it passed the test.

- Improving your process with reflection

My reflections

I tried almost all of the techniques above during the code challenges, except the rubber ducky method. I find googling is the most effient way of problem sovling. Chances are whatever problem we have, other people have already had it and had came up with a solution for the problem. Normally Google can solve 90 percent of the problems. If not, try to ask the question in a different way, it will solve a further 5% of the problems. As for the remaining 5 percent, ask for help for your specific question from other people, students, collegues, friends or forums.
